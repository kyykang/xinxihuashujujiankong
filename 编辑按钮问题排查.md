# 编辑按钮问题排查指南

## 问题描述
用户报告"现在没法点击编辑了"，编辑按钮无响应。

## 可能的原因

### 1. JavaScript 错误
- 检查浏览器控制台是否有 JavaScript 错误
- 按 F12 打开开发者工具，查看 Console 标签

### 2. Bootstrap 未加载
- 检查 Bootstrap JS 是否正确加载
- 网络问题可能导致 CDN 资源加载失败

### 3. 函数作用域问题
- `editTarget` 函数在 `{% block extra_js %}` 中定义
- 可能存在作用域问题

## 排查步骤

### 步骤 1: 检查浏览器控制台
1. 打开监控目标页面 (http://localhost:8080/targets)
2. 按 F12 打开开发者工具
3. 切换到 Console 标签
4. 点击编辑按钮
5. 查看是否有错误信息

### 步骤 2: 测试函数是否存在
在浏览器控制台输入：
```javascript
typeof editTarget
```
如果返回 "function"，说明函数已定义。

### 步骤 3: 手动调用函数
在浏览器控制台输入（假设有 ID 为 1 的目标）：
```javascript
editTarget(1)
```
查看是否能打开编辑模态框。

### 步骤 4: 检查 Bootstrap
在浏览器控制台输入：
```javascript
typeof bootstrap
```
如果返回 "undefined"，说明 Bootstrap 未加载。

## 临时解决方案

如果是 CDN 加载问题，可以：
1. 清除浏览器缓存（Cmd+Shift+R）
2. 检查网络连接
3. 尝试使用其他 CDN 源

## 已知问题

从代码检查来看：
- HTML 结构正确：`<button class="btn btn-sm btn-warning" onclick="editTarget({{ target.id }})">编辑</button>`
- JavaScript 函数存在：`function editTarget(id) { ... }`
- Bootstrap 已引入：`<script src="https://cdn.bootcdn.net/ajax/libs/bootstrap/5.1.3/js/bootstrap.bundle.min.js"></script>`
